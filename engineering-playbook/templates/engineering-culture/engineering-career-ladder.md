# Engineering Career Ladder & Competency Matrix

*This matrix defines exactly what is expected of an engineer at every level. It removes bias from promotions and provides a clear roadmap for career advancement.*

## 1. Core Competency Tracks
1.  **Technical Execution:** Writing code, architecture, testing, and debugging.
2.  **Delivery & Impact:** Shipping features, unblocking others, and meeting deadlines.
3.  **Communication & Leadership:** Mentoring, giving feedback, and leading cross-functional projects.

## 2. The Levels

### Level 1: Software Engineer I (Junior)
*   **Technical:** Writes code to spec. Needs guidance on complex logic. Learning the architecture.
*   **Delivery:** Can complete well-defined, isolated tasks within a sprint.
*   **Leadership:** Asks good questions. Absorbs feedback. Focuses on self-improvement.

### Level 2: Software Engineer II (Mid-Level)
*   **Technical:** Writes modular, well-tested code. Understands the broader architecture. Can debug production issues.
*   **Delivery:** Owns medium-sized features end-to-end. Predictable velocity.
*   **Leadership:** Mentors Juniors. Participates actively in code reviews.

### Level 3: Senior Software Engineer
*   **Technical:** Designs systems. Anticipates edge cases and performance bottlenecks. Masters their primary language.
*   **Delivery:** Capable of breaking down massive epics into sprint-sized tasks. Consistently ships high-impact work.
*   **Leadership:** A force multiplier. Elevates the entire team's code quality. Leads technical interviews.

### Level 4: Staff Engineer (Technical Lead)
*   **Technical:** Sets the technical direction for an entire domain. Authors Architecture Definition Documents (ADDs).
*   **Delivery:** Aligns technical strategy with business goals. Resolves cross-team technical dependencies.
*   **Leadership:** Mentors Senior engineers. Represents engineering to stakeholders and executives.

## 3. Promotion Process
*   Promotions are not based on tenure (time in seat).
*   To be promoted, an engineer must consistently demonstrate the behaviors of the *next* level for at least 3-6 months.
